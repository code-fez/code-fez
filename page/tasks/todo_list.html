Task: Build a To-Do List App

Requirements:

User Interface (UI):
  • Create a simple command-line interface (CLI) for the app.
  • Display a menu with options for adding tasks, viewing tasks, marking tasks as complete, and quitting the app.
Data Storage:
  • Use a list or dictionary to store the tasks.
  • Each task should have the following properties:
    • Task name
    • Due date (optional)
    • Priority level (e.g., high, medium, low)
    • Whether done or not done
Functionality:
  • Implement the following features:
    • Add Task: Allow users to add new tasks with a name, due date, and priority level.
    • View Tasks: Display all tasks along with their details (name, due date, priority).
    • Mark Task as Complete: Users can mark a task as completed.
    • Delete a Task that is selected: Users should be able to delete a task from the list if it is chosen.
    • Quit: Exit the app.
Programming Constructs:
  • Use appropriate programming constructs such as loops, conditionals, and functions.
  • Ensure clean and modular code.
Selection:
  • Use conditional statements (if/else) to handle user input and menu options.
Iteration:
  • Implement loops to display tasks and handle repeated actions (e.g., displaying the menu).
Functions:
  • Create functions for adding tasks, viewing tasks, and marking tasks as complete.
